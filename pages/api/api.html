Get Articles by Tag ID

URL: /api/article/byTag/:tag_id

Method: GET

Description: This endpoint retrieves all articles associated with a specific tag, identified by its tag_id. It allows users to filter articles based on tags for more targeted access to content.

Parameters

  • Path Parameter: :tag_id (string) – Required. The unique identifier of the tag whose associated articles you want to retrieve.

Request Example

This request fetches all articles with the tag ID "12345".

GET /api/article/byTag/12345

Responses

Success (200 OK)

A successful response will return a JSON object containing an array of articles associated with the specified tag_id. Each article includes details like title, content, author, and other relevant metadata.

{ "status": "success", "data": [ { "id": "article123", "title": "The Future of Computational Intelligence", "author": "John Doe", "published": "2024-08-11T10:00:00Z", "description": "This article explores the future of computational intelligence...", "shortDescription": "In this article...", "tags": ["computational", "intelligence"] }, { "id": "article124", "title": "Introduction to Quantum Mechanics", "author": "Jane Smith", "published": "2024-09-01T12:30:00Z", "description": "This article provides an introduction to quantum mechanics...", "shortDescription": "In this article...", "tags": ["physics", "quantum", "mechanics"] } ] }

404 Not Found

The requested tag ID does not exist or has no associated articles.

{ "status": "error", "message": "No articles found for the specified tag ID." }

Response Fields

  • status (string): Status of the response, either "success" or "error".
  • data (array): List of articles associated with the given tag ID.
    •  id (string): Unique identifier of the article.
    •  title (string): Title of the article.
    •  author (string): Name of the author who wrote the article.
    •  published (string): The date and time the article was published.
    •  description (string): The full description of the article.
    •  shortDescription (string): The short description of the article.
    •  tags (array): List of tags associated with the article.

Notes

  • Ensure that the tag_id provided in the request URL is valid and corresponds to a tag in the system.
  • If no articles are found for the given tag_id, a 404 response will be returned.

Get Global Tags

URL: /api/article/tags

Method: GET

Description: This endpoint retrieves a list of all global tags that are used across articles. Tags help categorize content and make it easier for users to search for articles based on specific topics or themes.

Parameters

  • No Parameters: This endpoint does not require any path or query parameters.

Request Example

This request fetches all available global tags.

GET /api/article/tags

Response

Success (200 OK)

A successful response will return a JSON object containing an array of tags used globally across all articles. Each tag includes a unique identifier, the name, and the description of the tag.

{ "status": "success", "data": [ { "id": "tag123", "name": "computational", "description": "Techniques and methods for solving complex problems using algorithms and computational systems." }, { "id": "tag124", "name": "intelligence", "description": "Exploration of smart systems and cognitive technologies that enable decision-making and problem-solving." }, { "id": "tag125", "name": "physics", "description": "Study of natural laws and phenomena governing matter, energy, and their interactions." } ] }

Response Fields

  • status (string): Status of the response, either "success" or "error".
  • data (array): List of global tags.
    •  id (string): Unique identifier of the tag.
    •  name (string): The name of the tag.
    •  description (string): The description of the tag.

Notes

  • The list of tags retrieved will include all tags that have been used in articles.
  • No additional parameters are needed for this request. The endpoint will return all available global tags.

Get All Authors

URL: /api/article/authors

Method: GET

Description: This endpoint retrieves a list of all authors who have contributed articles. It allows users to see the complete set of authors, including their basic information such as name and ID.

Parameters

  • No Parameters: This endpoint does not require any path or query parameters.

Request Example

This request fetches a list of all available authors.

GET /api/article/authors

Response

Success (200 OK)

A successful response will return a JSON object containing an array of authors. Each author includes a unique identifier, name, and additional metadata such as the number of articles they have written.

{ "status": "success", "data": [ { "id": "author123", "name": "John Doe", "article_count": 12 }, { "id": "author124", "name": "Jane Smith", "article_count": 8 }, { "id": "author125", "name": "Alice Johnson", "article_count": 15 } ] }

Response Fields

  • status (string): Status of the response, either "success" or "error".
  • data (array): List of authors who have contributed articles.
    •  id (string): Unique identifier of the author.
    •  name (string): Name of the author.
    •  article_count (integer): The total number of articles written by the author.

Notes

  • This endpoint returns a complete list of all authors in the system.
  • The article_count field gives the total number of articles each author has contributed.

Get Author by Author ID

URL: /api/article/author/:author_id

Method: GET

Description: This endpoint retrieves details about a specific author, identified by their author_id. It allows users to access detailed information about an individual author, including their name, bio, and the number of articles they have written.

Parameters

  • Path Parameter: :author_id (string) – Required. The unique identifier of the author whose details you want to retrieve.

Request Example

This request fetches details of the author with ID "67890".

GET /api/article/author/67890

Responses

Success (200 OK)

A successful response will return a JSON object containing details of the specified author, including their unique ID, name, bio, and article count.

{ "status": "success", "data": { "id": "author67890", "name": "Jane Smith", "bio": "Jane Smith is an expert in 3D modeling...", "article_count": 12 } }

404 Not Found

The requested author ID does not exist in the system.

{ "status": "error", "message": "Author not found for the specified ID." }

Response Fields

  • status (string): Status of the response, either "success" or "error".
  • data (object): Details of the specified author.
    •  id (string): Unique identifier of the author.
    •  name (string): Name of the author.
    •  bio (string): Short biography or description of the author.
    •  article_count (integer): The total number of articles written by the author.

Notes

  • Ensure that the author_id provided in the request URL is valid and corresponds to an author in the system.
  • If the author_id is invalid, a 404 response will be returned.
